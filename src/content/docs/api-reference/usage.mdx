`GET /v1/admin/usage` returns aggregated usage statistics for your account. No raw translation text is exposed — only counts, confidence distributions, and risk breakdowns.

For the full schema, see the [Interactive Reference](/api-reference/index/).

---

## Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `from` | ISO 8601 date | Start of the reporting window |
| `to` | ISO 8601 date | End of the reporting window |
| `domain` | string | Filter by clinical domain (optional) |
| `source_language` | string | Filter by source language (optional) |
| `target_language` | string | Filter by target language (optional) |
| `granularity` | `day` \| `week` \| `month` | Aggregation bucket (default: `day`) |

## Using usage data for monitoring

- **Elevated `by_risk.critical` rate** — if `critical` exceeds 2–3% of total, investigate whether the correct `domain` is being passed for the texts being translated.
- **Declining `confidence_p50`** — a drop in median confidence over time may indicate a language pair mismatch or domain mismatch in your integration.
- **`withheld_count` trends** — rising withheld counts are an early signal of translation quality issues for a specific language pair or domain.
