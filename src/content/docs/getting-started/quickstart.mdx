Get a working medical translation in **5 minutes**.

:::note[You'll need a test API key]
Tolka is in private launch. Request a test key (`sk_test_...`) via the early-access form at **[tolka.health](https://tolka.health)** — keys are issued the same day during business hours (Europe/Oslo).

Throughout this guide, replace `sk_test_YOUR_KEY_HERE` with your real key.

Test keys are rate-limited (see [Rate limits](/getting-started/rate-limits/)) and process the same translation pipeline as production keys — no behavioral differences.
:::

## Choose your path

**Path A — Zero tooling (CDN script tag).** Paste into any HTML file and open in a browser.

**Path B — npm project.** Recommended for real integrations.

---

## Path A: CDN (30 seconds)

Create `index.html` and open it in your browser:

```html
<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"><title>Tolka test</title></head>
<body>
  <tolka-translate
    api-key="sk_test_YOUR_KEY_HERE"
    source-lang="no"
    target-lang="en"
    domain="gp">
  </tolka-translate>

  <script
    src="https://cdn.tolka.health/sdk/web/latest/tolka.min.js"
    crossorigin="anonymous">
  </script>
</body>
</html>
```

You should see the `<tolka-translate>` widget render with a text input and translated output.

## Path B: npm

<Steps>
1. **Install the SDK**

   ```bash
   pnpm add @tolka/web-sdk
   # or: npm install @tolka/web-sdk
   ```

2. **Make your first translation**

   <Tabs>
   <TabItem label="TypeScript">
   ```typescript
   import { TolkaClient } from '@tolka/web-sdk';

   const tolka = new TolkaClient({
     apiKey: 'sk_test_YOUR_KEY_HERE',
     baseUrl: 'https://api.tolka.health',
   });

   const result = await tolka.translate({
     text: 'Har du smerter i brystet?',
     source_language: 'no',
     target_language: 'en',
     mode: 'standard',
     context: { domain: 'emergency' },
   });

   console.log(result.translation);   // "Do you have chest pain?"
   console.log(result.confidence);    // 0.94
   console.log(result.risk);          // "low"
   ```
   </TabItem>
   <TabItem label="cURL">
   ```bash
   curl -X POST https://api.tolka.health/v1/medical/translate \
     -H "Authorization: Bearer sk_test_YOUR_KEY_HERE" \
     -H "Content-Type: application/json" \
     -d '{
       "text": "Har du smerter i brystet?",
       "source_language": "no",
       "target_language": "en",
       "mode": "standard",
       "context": { "domain": "emergency" }
     }'
   ```
   </TabItem>
   <TabItem label="Python">
   ```python
   import requests

   response = requests.post(
       "https://api.tolka.health/v1/medical/translate",
       headers={"Authorization": "Bearer sk_test_YOUR_KEY_HERE"},
       json={
           "text": "Har du smerter i brystet?",
           "source_language": "no",
           "target_language": "en",
           "mode": "standard",
           "context": {"domain": "emergency"},
       },
   )
   data = response.json()
   print(data["translation"])   # "Do you have chest pain?"
   print(data["confidence"])    # 0.94
   print(data["risk"])          # "low"
   ```
   </TabItem>
   </Tabs>

3. **Verify your setup**

   A successful response looks like this:

   ```json
   {
     "translation": "Do you have chest pain?",
     "back_translation": "Har du brystsmerter?",
     "confidence": 0.94,
     "risk": "low",
     "flags": [],
     "alternatives": [],
     "verified": true,
     "metadata": {
       "latency_ms": 2840,
       "model": "gpt-4-turbo-2024-04-09",
       "request_id": "req_01HX..."
     }
   }
   ```

   Check `verified: true` and `risk: "low"`. You're ready to integrate.

</Steps>

## Troubleshooting

| Error | Code | Fix |
|-------|------|-----|
| `INVALID_API_KEY` | `401` | Verify the key is prefixed `sk_test_` or `sk_live_`. No extra whitespace. |
| `RATE_LIMIT_EXCEEDED` | `429` | Wait for the `Retry-After` header value (seconds) before retrying. Test keys default to 60 req/min. |
| `SERVICE_UNAVAILABLE` | `503` | Tolka is temporarily down. Check [the support page](/about/support/) and retry with exponential backoff. |
| `TEXT_TOO_LONG` | `400` | Text exceeds 2,000 characters. Split into shorter segments at sentence boundaries. |
