GP consultations involve multiple back-and-forth exchanges. Using sessions ensures each exchange benefits from conversation context, and avoids passing `domain`, `source_lang`, and `target_lang` on every call.

---

## Consultation flow with sessions

<Steps>
1. **Create a session when the consultation starts**

   ```typescript
   import { TolkaClient } from '@tolka/web-sdk';

   const tolka = new TolkaClient({ apiKey: process.env.TOLKA_API_KEY! });

   // Called when the GP opens the patient consultation screen
   const session = await tolka.sessions.create({
     domain: 'gp',
     sourceLang: 'ar',  // patient's language
     targetLang: 'nb',  // clinician's language
     ttlMinutes: 60,
   });

   // Store in component state — not in localStorage
   const sessionToken = session.token;
   ```

2. **Translate each exchange using the session token**

   ```typescript
   // GP types a question; translate to patient's language
   const result = await tolka.safeTranslate({
     text: 'Har du hatt feber de siste dagene?',
     sessionToken,
     // source_lang, target_lang, domain inferred from session
   });
   ```

3. **Handle risk per exchange**

   ```typescript
   function handleExchange(result: SafeTranslateResponse) {
     if (result.withheld) {
       return showRefusal(result.flags);
     }
     if (result.risk === 'high') {
       return showAcknowledgmentPrompt(result);
     }
     return showTranslation(result.translation, result.risk);
   }
   ```

4. **Close the session when the consultation ends**

   Sessions expire automatically after `ttl_minutes`. You don't need to explicitly close them, but creating a new session for each consultation is best practice — it prevents context bleeding between patients.

</Steps>

## Example consultation UI pattern

```tsx
function ConsultationTranslation({ sessionToken }: { sessionToken: string }) {
  const [exchanges, setExchanges] = useState<Exchange[]>([]);

  async function onSubmit(text: string) {
    const result = await tolka.safeTranslate({ text, sessionToken });

    setExchanges((prev) => [
      ...prev,
      {
        source: text,
        translation: result.withheld ? null : result.translation,
        backTranslation: result.back_translation,
        risk: result.risk,
        flags: result.flags,
        withheld: result.withheld,
        timestamp: new Date(),
      },
    ]);
  }

  return (
    <div>
      <ExchangeHistory exchanges={exchanges} />
      <TranslationInput onSubmit={onSubmit} />
    </div>
  );
}
```

## Session TTL guidance

| Consultation type | Recommended TTL |
|-------------------|----------------|
| Routine GP appointment | 30 minutes |
| Extended specialist consultation | 90 minutes |
| Walk-in/urgent care | 20 minutes |
| Telehealth | 60 minutes |

Set `ttl_minutes` conservatively — expired sessions return `SESSION_EXPIRED` and your UI should create a new one silently, not surface the error to the clinician.
