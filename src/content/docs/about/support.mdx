## Support channels by plan

| Plan | Channel | Response time |
|------|---------|---------------|
| Starter | [support@tolka.health](mailto:support@tolka.health) | 2 business days |
| Pro | Priority email + [support@tolka.health](mailto:support@tolka.health) | 1 business day |
| Enterprise | Dedicated Slack channel + 24/7 email | 4 hours |

All plans also have access to [GitHub Discussions](https://github.com/tolka-health/docs/discussions) for community support. See [Pricing](/about/pricing/) for a full plan comparison.

## Before opening a ticket

Include these in every support request — they allow us to pull the exact request from logs immediately:

1. **`metadata.request_id`** from the affected API response (e.g., `req_01HX4K2MN7...`)
2. The **endpoint** called (`/translate`, `/safe-translate`, etc.)
3. The **error code** if applicable (e.g., `TRANSLATION_FAILED`)
4. Your **key prefix** (e.g., `sk_test_` or `sk_live_`) — never the full secret
5. The **approximate time** of the request (UTC)

Do **not** include source text, translations, or any patient-identifiable information in support tickets.

## Security vulnerabilities

Report security issues to [security@tolka.health](mailto:security@tolka.health). Do not open a public GitHub issue for security vulnerabilities.

We aim to acknowledge security reports within 24 hours and provide a remediation timeline within 72 hours.
